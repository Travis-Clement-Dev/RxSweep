# RxSweep Day 2 Implementation Plan — Web App

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the local web app that makes Dana's journey interactive — upload → live progress → severity-first triage dashboard → verify with citations → grounded chat — served by one command with the built frontend bundled inside the Python package.

**Architecture:** FastAPI backend (`rxsweep serve`) wraps the existing Day 1 engine unchanged; sweeps run in a background thread whose progress is observed by tapping the same audit-event stream that feeds the JSONL log (one event pipeline, two consumers — governance and UX from the same source). Frontend is Vite + React + TS + Tailwind + shadcn/ui-style components, custom-themed to the established teal clinical tokens, built to static assets that FastAPI serves from inside the package — peers never need Node.

**Tech Stack:** FastAPI + uvicorn + python-multipart (new deps); Vite/React/TS/Tailwind v4 (dev-only, in `web/`); build output committed to `src/rxsweep/webapp/static/`.

## Global Constraints

- Day 1 engine modules are consumed, not modified (exception: extracting the pipeline out of `cli.py` into a reusable function).
- Every AI interaction (including chat) logs prompt + completion verbatim to the run's `audit.jsonl` (spec: Governance pack).
- Chat is grounded ONLY in the selected run's findings; the system prompt forbids answering beyond them; every drug claim carries its [n] citation (spec: UX journey stage 5).
- Scope banner (informational, pharmacist verifies) and openFDA disclaimer appear in the app shell, not just the HTML report.
- Design tokens identical to `report.html.j2` (`--accent:#0E7C6B` family + severity status colors, light and dark). No default-shadcn look: custom theme. Severity colors reserved for severity, never decoration (dataviz rules).
- WCAG 2.1 AA baseline: keyboard operability, visible focus, labels/roles on interactive elements, contrast per tokens.
- User-facing copy: plain regulatory register, no em dashes, no markdown leakage.
- Local-first: server binds 127.0.0.1 by default.
- Conventional commits, one logical change per task.

**Files:** Create `src/rxsweep/pipeline.py`; Modify `src/rxsweep/cli.py`; Test `tests/test_pipeline.py`

**Interfaces (produces):**

- `SweepResult` (pydantic): `run_id: str`, `run_dir: Path`, `findings: list[Finding]`, `quarantined: list[QuarantinedRow]`, `manual_review: list[Candidate]`, `unchecked: list[str]`, `summary: str | None`, `meta: dict`, `tiers: dict[str, int]`, `report_path: Path`
- Every audit event is also forwarded to `on_progress` (single tap point: wrap the `AuditLog.event` call).
- `cli.py check` becomes a thin wrapper: call `run_sweep`, print, exit 1 on critical. Existing e2e tests must pass unchanged.

Steps: failing test (run_sweep on fixture-mocked endpoints returns SweepResult with report on disk and progress callback invoked with `run_start`…`run_end`) → implement → all tests green → commit `refactor: extract reusable sweep pipeline from CLI`.

### Task D2-2: FastAPI backend — sweep runs with live progress

**Files:** Create `src/rxsweep/webapp/__init__.py`, `src/rxsweep/webapp/server.py`; Test `tests/test_server.py`

**Interfaces (produces):** `create_app() -> FastAPI` with:
- `POST /api/sweeps` (multipart CSV upload, options `months_back`, `use_ai`) → `202 {"run_id": ...}`; sweep executes in `threading.Thread`; progress state per run updated from `on_progress` events: `{phase, items, fda_requests, ai_calls, done, error}`.
- `GET /api/sweeps/{run_id}` → progress while running; when done: full result JSON (findings serialized via pydantic, tiers, quarantined, manual_review, unchecked, summary, meta).
- `GET /api/sweeps/{run_id}/report` → the generated report.html (FileResponse).
- Unknown id → 404. Upload without CSV → 422. Undetectable columns → surfaced as `error` in progress state, not a 500.
- Runs stored under `runs/` exactly as the CLI does (same audit trail).

Tests: TestClient + respx-mocked openFDA; poll until done; assert findings JSON shape, 404 path, bad-CSV error path. AI off in tests (no key).

Commit: `feat: FastAPI backend with background sweeps and live progress`.

### Task D2-3: Grounded chat endpoint

**Files:** Modify `src/rxsweep/webapp/server.py`; Create `src/rxsweep/chat.py`; Test `tests/test_chat.py`

**Interfaces (produces):**
- `chat_reply(findings: list[Finding], history: list[dict], question: str, audit: AuditLog, model: str | None = None) -> str` in `chat.py`. System prompt: pharmacist-facing; answer ONLY from the numbered findings context; every drug claim cites [n]; if asked beyond the findings, say the sweep doesn't cover it; regulatory register, plain text, no em dashes. API failure → polite unavailable message (never 500), logged `ai_unavailable`.
- `POST /api/sweeps/{run_id}/chat` `{question, history}` → `{reply}`; 409 if run not finished; requires `ANTHROPIC_API_KEY` (else 503 with clear message). Chat prompts/replies audit-logged verbatim to the run's audit.jsonl (`kind: ai_request/ai_response, stage: chat`).

Tests: mocked Anthropic — grounding context contains findings citations; failure path returns unavailable message; endpoint auth-missing path.

Commit: `feat: run-grounded cited chat endpoint`.
