Federal clarity, contemporary craft

The design system for RxSweep's React Aria exploration. Every choice below is anchored to Dana's journey (Prepare → Run → Triage → Verify → Act) and aimed at the known failure of government-enterprise software: trustworthy but a decade old, or modern but unserious. RxSweep is a regulatory instrument with the polish of the best modern tools. This page is written in the system it defines.

1 · Principles

  • Worklist first. The hero of every screen is what the pharmacist does next, never what the software produced.
  • Severity is state, not decoration. Status colors are reserved for severity meaning; the teal accent never signals danger, danger colors never decorate.
  • One accent, neutral everything else. Teal is the only brand-carrying color and appears sparsely (buttons, focus, links, the band rule). Surfaces are never brand-tinted; identity comes from the accent plus typography and document structure.
  • Document language. Square corners, rules, registers, exhibits: artifacts read as instruments of record, on screen and in print.
  • AI is labeled, cited, summoned. Model output appears attached to objects or on demand, always with citations and its cost on the meter.
  • Accessibility is the floor, not a feature. WCAG 2.1 AA contrast (verified below, computed live), full keyboard operability via React Aria, visible focus everywhere.

2 · Color tokens

Research grounding (primary sources)

  • USWDS: "start in black and white"; color is progressive enhancement, never the sole carrier of meaning; contrast by grade arithmetic. (designsystem.digital.gov)
  • NHS service manual: pages sit on light grey, not white (glare + British Dyslexia Association guidance); white is reserved for surfaces that must stand out; color only in strict semantic roles; brand color exists for trust recognition. Notably, the NHS's own primary action color is green. (service-manual.nhs.uk)
  • VA.gov design system: primitives strictly from USWDS; a single trust-color predominant; semantic tokens over decorative ones. (design.va.gov)

How this shaped the palette: surfaces are true neutrals (no brand-tinted backgrounds; none of the reference systems tint theirs), the light ground is NHS-style grey with white reserved for panels, and teal remains the single sparse accent: the blue-green trust family, at home in healthcare (NHS green), and deliberately distinct from every blue EHR the audience stares at all day. Severity colors follow the shared semantic convention: red urgent, amber caution, green normal/success.

3 · Typography

Public Sans (the U.S. federal open-source face) with Helvetica fallback; production bundles the woff2. Monospace (system SF Mono/Menlo) is reserved for identifiers: NDCs, run ids, citations, file names. No decorative faces in the app; Georgia serif belongs to the memo export only.

4 · Space & density

Base unit 4px. Density targets an instrument, not a brochure: table rows 32–36px tall, queue rows 40px, panel padding 18–22px, section gaps 40px. One content column max-width 1280px with rail 208px and dock 300px. Corners: 0px everywhere (square is the identity); the only circles are severity dots and the radar sweep.

6 · Interaction states

  • Focus: 2px teal outline, 2px offset, on every interactive element; table rows use inset 3px left bar + tinted row. Focus is never suppressed.
  • Hover: soft accent tint on rows and rail items; buttons brighten 8%. Hover never carries meaning alone.
  • Pressed/selected: aria-pressed/selected drives the visual (React Aria state attributes), not click-order CSS.
  • Motion: one signature animation (the radar sweep) plus row-flash on citation jump; everything else is instant. prefers-reduced-motion disables all.
  • Dark theme: same tokens re-derived, verified by the computed ratios above; never an inversion filter.

7 · Writing register

  • FDA drug-safety-communication register: short declarative sentences, active voice, standard terminology (recall classification, shortage status, presentation).
  • No em dashes. No markdown syntax in generated text. Plain text paragraphs.
  • Every AI-drafted element is labeled and cited; scope banner on every surface: informational tool, pharmacist verifies, not clinical advice.
  • Buttons say exactly what happens: "Run sweep", "Open FDA source record", "Brief me for the huddle".
  • openFDA's disclaimer travels verbatim with every artifact.

8 · Generic-AI patterns we reject

Don't

  • Rounded cards with drop shadows as the default container
  • AI prose as the dashboard hero
  • Pill-shaped colored badges for every attribute
  • Purple/indigo gradients, glassmorphism, emoji as section markers
  • Centered marketing-style layouts for working screens
  • Severity colors reused as decoration or accents
  • Brand-tinted surfaces or a second accent hue: backgrounds are never teal-cast (guide v1 made exactly this mistake; the USWDS/NHS/VA research corrected it, see D9)

Do

  • Square panels, hairline borders, document rules
  • Verb-led worklist as the hero; AI summoned on demand
  • Typographic state (small-caps + dot) for severity
  • Neutral surfaces carry the work; exactly one accent color (teal) carries the brand, used sparsely: buttons, focus, links
  • One signature motion (the radar sweep), tabular numerals
  • Left-aligned instrument density with a rail and a dock
  • Provenance everywhere: citations, doc links, audit trail, cost meter
